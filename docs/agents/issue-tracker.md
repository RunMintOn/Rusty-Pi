# Issue tracker: Local Markdown

Issues for this repo live as markdown files in the repo root.

## Conventions

- The spec is `SPEC.md` at the repo root
- Tickets are `tickets.md` at the repo root, in dependency order
- Triage is not used (solo project, no incoming issues)
- Work progresses through the `tickets.md` frontier one ticket at a time

## When a skill says "publish to the issue tracker"

Update `tickets.md` or create a new ticket file in the repo root.

## When a skill says "fetch the relevant ticket"

Read the file at the referenced path.
